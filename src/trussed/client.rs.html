1//! # Client interface for applications.
2//!
3//! The API methods (such as `GenerateKey`, `Sign`, `Verify`,...) are implemented by a variety
4//! of mechanisms (such as `Ed255`, `X255`, `Chacha8Poly1305`, `HmacSha256`,...).
5//!
6//! The `ClientImplementation` structure in this module offers only one general `request` method:
7//! ```ignore
8//! pub fn request<'c, T: From<Reply>>(&'c mut self, req: impl Into<Request>)
9//!   -> ClientResult<'c, T, Self>
10//! ```
11//!
12//! For convenience, the `Client` trait expands the API methods, keeping the mechanism general,
13//! e.g.:
14//! ```ignore
15//! // use trussed::Client as _;
16//! fn sign<'c>(&'c mut self,
17//!   mechanism: Mechanism,
18//!   key: KeyId,
19//!   data: &[u8],
20//!   format: SignatureSerialization
21//! ) -> ClientResult<'c, reply::Sign, Self>;
22//! ```
23//!
24//! For further convenience, each mechanism has a corresponding trait of the same name, e.g.,
25//! `Ed255`, which also specializes the mechanism, e.g.
26//! ```ignore
27//! // use trussed::client::Ed255 as _;
28//! fn sign_ed255<'c>(&'c mut self, key: &KeyId, message: &[u8])
29//!   -> ClientResult<'c, reply::Sign, Self>
30//! ```
31//!
32//! Pick your poison :)
33//!
34//! # Details
35//!
36//! The lower-level workings of `ClientResult` are currently a hand-rolled / semi-horrible
37//! pseudo-`Future` implementation; this will likely be replaced by a proper `core::future::Future`
38//! with something like the [direct-executor](https://github.com/dflemstr/direct-executor).
39//!
40//! The lifetimes indicate that the `ClientResult` takes ownership of the unique reference
41//! to the client itself for the length of its own lifetime. That is, once the call to Trussed
42//! completes (success or failure), there is no use for the `ClientResult` anymore, so due to
43//! lexical lifetimes, the `ClientImplementation` can be used again.
44//!
45//! What does always happen is that each client has an Interchange with the service, in which
46//! it places the `api::Request` (a Rust enum), and then uses the `Syscall` implementation, to
47//! trigger processing by the Trussed service.
48//!
49//! In practice, in embedded Syscall is implemented by pending a hardware interrupt for the
50//! service, which runs at a higher interrupt priority. For PC testing, the service itself
51//! has a Syscall implementation ("call thyself"). In both cases, the caller is blocked until
52//! processing completes.
53//!
54//! All the same, to unpack the "result" it is suggested to use the `syscall!` macro, which
55//! returns the `Reply` corresponding to the `Request`. Example:
56//! ```ignore
57//! let secret_key = syscall!(client.generate_x255_secret_key(Internal)).key;
58//! ```
59//!
60//! This `syscall!` can fail (by panicking) in two ways:
61//! - logic error: clients are only allowed to make one syscall to the Trussed service at once,
62//!   then they must wait for a response. By the above, this case cannot happen in practice.
63//! - processing error: some methods are naturally fallible; for example, a public key that is
64//!   to be imported via `Deserialize` may be invalid for the mechanism (such things are always checked).
65//!
66//! In this second case (probably in all cases when programming defensively, e.g. one possible
67//! `trussed::error::Error` is `HostMemory`, which means out of RAM), the `try_syscall!` macro
68//! should be used instead, which does not unwrap the inner Result type.
69//!
70//! In terms of the `Result<FutureResult<'c, T, C>, ClientError>` return type of the `Client::request`
71//! method, the outer `Result` corresponds to the logic error (see `trussed::client::ClientError`)
72//! for possible causes.
73//!
74//! The processing error corresponds to the `Result<From<Reply, trussed::error::Error>>` which is
75//! the `Ready` variant of the `core::task::Poll` struct returns by the `FutureResult`'s `poll` method.
76//! Possible causes are listed in `trussed::error::Error`.
77//!

120/// The client implementation client applications actually receive.
121pub struct ClientImplementation<'a, S, D = CoreOnly> {
122    // raw: RawClient<Client<S>>,
123    syscall: S,
124
125    // RawClient:
126    pub(crate) interchange: TrussedRequester<'a>,
127    pub(crate) interrupt: Option<&'static InterruptFlag>,
128    // pending: Option<Discriminant<Request>>,
129    pending: Option<u8>,
130    _marker: PhantomData<D>,
131}

160impl<S, E> PollClient for ClientImplementation<'_, S, E>
161where
162    S: Syscall,
163{
164    fn poll(&mut self) -> Poll<Result<Reply, Error>> {
165        match self.interchange.take_response() {
166            Some(reply) => {
167                // #[cfg(all(test, feature = "verbose-tests"))]
168                // println!("got a reply: {:?}", &reply);
169                match reply {
170                    Ok(reply) => {
171                        if Some(u8::from(&reply)) == self.pending {
172                            self.pending = None;
173                            Poll::Ready(Ok(reply))
174                        } else {
175                            // #[cfg(all(test, feature = "verbose-tests"))]
176                            info!(
177                                "got: {:?}, expected: {:?}",
178                                Some(u8::from(&reply)),
179                                self.pending
180                            );
181                            Poll::Ready(Err(Error::InternalError))
182                        }
183                    }
184                    Err(error) => {
185                        self.pending = None;
186                        Poll::Ready(Err(error))
187                    }
188                }
189            }
190            None => {
191                debug_assert_ne!(
192                    self.interchange.state(),
193                    interchange::State::Idle,
194                    "requests can't be cancelled"
195                );
196                Poll::Pending
197            }
198        }
199    }
200
201    // call with any of `crate::api::request::*`
202    fn request<Rq: RequestVariant>(&mut self, req: Rq) -> ClientResult<'_, Rq::Reply, Self> {
203        // TODO: handle failure
204        // TODO: fail on pending (non-canceled) request)
205        if self.pending.is_some() {
206            return Err(ClientError::Pending);
207        }
208        // since no pending, also queue empty
209        // if !self.ready() {
210        //     return Err(ClientError::Fulle);
211        // }
212        // in particular, can unwrap
213        let request = req.into();
214        self.pending = Some(u8::from(&request));
215        self.interchange.request(request).unwrap();
216        self.syscall.syscall();
217        Ok(FutureResult::new(self))
218    }
219
220    fn interrupt(&self) -> Option<&'static InterruptFlag> {
221        self.interrupt
222    }
223}
